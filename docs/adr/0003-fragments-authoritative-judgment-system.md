# ADR-0003: Fragments As Authoritative Judgment System

## Status

Accepted

## Context

The user needs low-friction capture for ideas, observations, links, and unstructured input. The wiki must stay dense and theoretical, not become a second inbox.

## Decision

`fragments/` is the authoritative system for unresolved input and judgment.

Fragments are judged, logged, archived when useful for audit, and then removed from active flow. Wiki pages do not preserve fragment backlinks.

## Consequences

The wiki contains promoted theory, not fragment provenance.

The fragment archive and log support audit, but they are not theory sources.
