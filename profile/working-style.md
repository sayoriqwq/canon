# Working Style

- 当系统设计不清楚时，回到用户的核心目标和 feedback loop 来挑战方案。
- 相比直接增加容器或分类，更偏好 first-principles reasoning。
- 当稳定用户偏好出现时，先说明它值得进入 profile，再更新 profile。
- v0 设计应保持低摩擦；如果结构本身会训练未来 agent 行为，则同时对齐目标结构。
- 区分 repository architecture 决策和软件工程理论，不要混在一起。
