# Canon 术语表

## 词条

`fragment`
: 未裁决的输入，例如想法、观察、问题、引用、直觉或非结构化笔记。fragment 被裁决后会从活跃流程中消亡。

`raw`
: 被保存的原始材料。raw 用于来源保真和后续检查，不用于解释。

`source note`
: `wiki/sources/` 下的页面，用来消化 raw source，并说明它对理论系统的贡献。

`wiki asset`
: `wiki/` 下的消化后理论资产，包括 source note、concept、claim、pattern、example、synthesis、question。

`concept`
: 稳定的理解对象，例如 state、boundary、runtime、tool use、memory、type modeling。

`claim`
: 可争议的判断，可以被支持、挑战、修订或退休。

`pattern`
: 可复用的结构或实践，可以指导设计或实现。

`example`
: 用来说明 concept、claim 或 pattern 的具体实例。

`synthesis`
: 由 sources、concepts、claims、patterns、examples、questions 连接后形成的当前高层理解。

`pin anchor`
: raw source 的轻量来源锚点，记录 upstream、pinned ref、本地投影和验证方式。它不是完整 vendor pin，也不替代 upstream authority。
